◆ alloc_output_buffer_fp

wiced_codec_data_transfer_api_t::alloc_output_buffer_fp

alloc_buffer_fp is the playback system's output buffer allocation method.

This function must be typically provided by the underlying technology. For example, BT/A2DP will provide its own alloc_buffer_fp, while Airplay would have its own. This
must be provided when <codec_if_instance>->init() is invoked.

◆ read_encoded_data_fp

wiced_codec_data_transfer_api_t::read_encoded_data_fp

read_encoded_data_fp is used to read in encoded data from the input queue.

As in alloc_buffer_fp, this function
must be provided by the underlying playback system as an argument to <codec_if_instance> -> init(). Note that the method name is from the viewpoint of the codec; the codec would need this function to read data from the system using the codec.

◆ write_decoded_data_fp

wiced_codec_data_transfer_api_t::write_decoded_data_fp

write_decoded_data_fp is used to write decoded data back to the playback system.

As in alloc_buffer_fp, this function
must be provided by the underlying playback system in <codec_if_instance> -> init().
